# 医宝 MVP：云端接口整体架构设计

## 1. 目标

为了让当前 Expo MVP 在面试和后续落地中具备“可扩展成真实产品”的说服力，云端层至少承担 4 类职责：

- 大模型对话编排：接收用户输入，拼接 system prompt，返回结构化消息卡片
- 就诊状态管理：记录从分诊、挂号、到院、缴费、结果、取药到诊后提醒的状态流转
- 场景主动推流：根据 LBS 与就诊状态触发主动提醒
- 亲属代办 / 志愿者协同：同步代办权限与人工兜底请求

## 2. 推荐最小技术方案

- 客户端：Expo + React Native
- BFF / API 层：Vercel Functions 或 Supabase Edge Functions
- 数据库：Supabase Postgres
- 模型调用：后端代理大模型，前端不直连
- 事件日志：存 `session_id`、`prompt_version`、`encounter_status`、`scene_trigger`

## 3. 最小接口清单

### `POST /chat`

用途：
- 接收用户自由输入
- 拼接 system prompt 与上下文
- 返回结构化消息列表

最小请求字段：
- `sessionId`
- `prompt`
- `symptom`
- `preferredTime`
- `expertPreference`
- `encounterStatus`
- `delegationMode`
- `appointment`
- `location`

最小响应字段：
- `promptVersion`
- `messages`
- `nextStatus`

### `POST /encounters/:id/status`

用途：
- 更新就诊主状态

关键状态：
- `SYMPTOM_INTAKE`
- `TRIAGED`
- `APPOINTMENT_CONFIRMED`
- `ON_THE_WAY`
- `ARRIVED_HOSPITAL`
- `WAITING_PAYMENT`
- `PARTIAL_RESULT_READY`
- `READY_FOR_PHARMACY`
- `CARE_PLAN_READY`

### `POST /scene/push`

用途：
- 根据位置与状态生成主动卡片

典型触发：
- 到院后推送“先去分诊台报到”
- 部分检查已出时推送“带着结果回科室”
- 处方同步后推送“去 1 楼 3 号窗口取药”

### `POST /delegation/invite`

用途：
- 发起亲属代办
- 返回代办人身份、权限和通知对象

### `POST /volunteer/request`

用途：
- 发起志愿者兜底
- 返回集合点、ETA、说明

## 5. 面试中可以这样表述

- 前端采用单页消息流承接用户就医主链路
- 云端以 BFF 方式统一承接模型调用、状态流转与主动推流
- 模型输出不是直接渲染文案，而是返回结构化消息对象，再由客户端渲染为聊天气泡和结果卡片
- LBS 与就诊状态共同作为触发条件，决定是否主动推送“到院提醒、缴费、结果、取药”等动作卡
