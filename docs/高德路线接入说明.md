# 高德路线接入说明

当前 MVP 已经把路线卡拆成两层：

1. `lib/routeService.ts`
   - 负责返回路线规划结果
   - 负责生成外部导航跳转链接
   - 现在使用的是 mock 路线数据

2. `components/LbsMapCard.tsx`
   - 只负责把路线数据显示成地图卡
   - 不关心数据来自 mock 还是高德真接口

仓库里现在还多了一层可直接部署的接口草稿：

3. `api/route/plans.ts` 与 `api/route/indoor.ts`
   - 采用 Vercel Functions 结构
   - 当前返回 mock 路线结果
   - 后续可以在这里替换成高德真请求

## 当前已经支持的能力

- 打车 / 公交地铁 / 自驾三种路线模式
- 路线步骤展示
- 院内二级带路卡
- 外部导航按钮

## 后续替换成高德真接口的最短路径

1. 在服务端新增路线接口
   - `POST /route/plans`
   - `POST /route/indoor`
   - 输入：起点、终点、路线方式 / 院内目标点
   - 输出：ETA、距离、换乘步骤、下车点、停车提示 / 院内带路步骤

2. 前端保留 `LbsMapCard` 不动
   - 只替换 `buildRoutePlans()`
   - 把 mock 数据改成服务端返回值

3. 外部导航保持 `openExternalNavigation()`
   - 当前是导航跳转适配层
   - 后续可以改成更正式的高德 URI API 参数

## 你现在最需要改的文件

- `lib/routeService.ts`
- `app/index.tsx`

## 当前前端已经支持的接口形状

如果你配置了环境变量：

```bash
EXPO_PUBLIC_ROUTE_API_BASE_URL=https://your-api.example.com
```

前端会优先请求下面两个接口。

### 1. 室外路线

`POST /route/plans`

请求体示例：

```json
{
  "origin": {
    "lat": 30.2741,
    "lng": 120.1551,
    "zone": "commuting",
    "label": "前往医院途中",
    "distanceToHospitalMeters": 1200
  },
  "destination": {
    "hospitalId": "zju-1",
    "hospitalName": "浙大一院",
    "lat": 30.2575,
    "lng": 120.1774
  }
}
```

返回体示例：

```json
{
  "plans": {
    "taxi": {
      "mode": "taxi",
      "label": "打车前往",
      "eta": "18 分钟",
      "distanceLabel": "距医院约 1.2 km",
      "summary": "推荐在门诊楼东侧临停区下车。",
      "steps": [
        { "key": "1", "kind": "taxi", "title": "呼叫网约车", "detail": "预计 3 分钟上车" }
      ],
      "externalLabel": "打开高德导航",
      "externalUrl": "https://uri.amap.com/navigation?..."
    }
  }
}
```

### 2. 院内带路

`POST /route/indoor`

请求体示例：

```json
{
  "hospital": {
    "id": "zju-1",
    "name": "浙大一院",
    "lat": 30.2575,
    "lng": 120.1774
  },
  "department": "呼吸内科",
  "focus": "triage"
}
```

返回体示例：

```json
{
  "plan": {
    "mode": "taxi",
    "label": "分诊台",
    "eta": "步行 2 分钟",
    "distanceLabel": "院内步行约 140 m",
    "summary": "到院后优先去分诊台报到。",
    "steps": [
      { "key": "1", "kind": "indoor", "title": "乘电梯到 3 楼", "detail": "沿标识前往分诊台" }
    ],
    "externalLabel": "查看院内导视",
    "externalUrl": "https://uri.amap.com/navigation?..."
  }
}
```

## 回退策略

- 配了 `EXPO_PUBLIC_ROUTE_API_BASE_URL` 就优先请求云端
- 请求失败或返回不完整时，自动回退到本地 mock 路线
- 所以当前 demo 不会因为接口没接好而跑不起来

## 建议的真实接入顺序

1. 先接公交 / 地铁 / 自驾路径规划
2. 再接当前位置定位
3. 再做到院状态触发
4. 最后再考虑原生地图 SDK
